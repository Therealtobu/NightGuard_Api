Protection Layers

Every layer blocks
a different attack.

Custom VM Bytecode

Source compiles to a proprietary instruction set. Only encrypted bytecode is in the output — executed by an embedded VM interpreter.

Opcode Aliasing

Every logical opcode has 1–3 aliases. Values shuffle per build — pattern-matching deobfuscators fail completely.

Dual-Layer Encryption

Bytecode passes through rolling-XOR cipher then a 32-byte key-block XOR. Both must be reversed in the correct order.

Opaque Predicates

Always-true/false conditions that look dynamic. Dead code blocks inject noise through automated analysis tools.

Control Flow Flattening

if/else and loops convert to state-machine dispatch. Logical program structure is completely destroyed for static analysis.

Per-Build Randomization

Same source → unique output every time. Different opcode values, different keys, different junk injection each run.

FAQ

Common questions.

How is this different from loadstring obfuscators?
Most obfuscators wrap code in loadstring() with base64 or XOR encoded strings. NightGuard V2 compiles to a custom bytecode format and runs it through an embedded VM interpreter — there is no Lua source in the output at all, only encrypted binary instructions.
Will the protected script still run correctly?
Yes. The NightGuard VM faithfully implements Lua 5.1 semantics — functions, closures, tables, varargs, string operations, and global access all behave identically to the original.
Does it support Roblox Luau?
Yes. The VM runtime is written in Lua 5.1 compatible code with a bitwise XOR shim that works across standard Lua 5.1, LuaJIT, and Roblox Luau environments.
What is Custom Seed?
The seed controls all randomization — opcode shuffling, junk injection, and encryption keys. With the same seed, the same source always produces the same output. Leave it blank for a random seed each time.
Is there a script size limit?
The web tool accepts scripts up to 512 KB. Obfuscation has a 60-second server-side timeout. For very large scripts, use the CLI locally: python cli.py input.lua output.lua
