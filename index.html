Protection Layers

Every layer makes
reversal harder.

NightGuard V2 stacks 10 independent protection passes, each targeting a different analysis vector.

Custom VM Bytecode

Source compiles to a proprietary instruction set. The original Lua is never present — only encrypted bytecode executed by an embedded VM interpreter.

Opcode Aliasing

Every logical opcode has 1–3 aliases with the same behavior. Opcode numbers shuffle per build — pattern-matching deobfuscators break completely.

Rolling-XOR Strings

Each byte's cipher key depends on the previous encrypted byte. Decrypted only inside the VM — never visible as plaintext in the output.

Control Flow Flattening

if/else blocks convert to state-machine dispatch loops. Logical program flow is completely destroyed from a static-analysis perspective.

Opaque Predicates

Conditions that always evaluate to true/false but look dynamic. Combined with dead code blocks that inject noise through analysis tools.

Dual-Layer Encryption

Serialized bytecode passes through rolling-XOR cipher then a 32-byte key-block XOR. Both layers must be reversed before any execution is possible.

FAQ

Common questions.

How is this different from other Lua obfuscators?
Most obfuscators wrap code in loadstring() with encoded strings. NightGuard V2 compiles to custom bytecode and executes through an embedded VM — there is no Lua source in the output, only encrypted binary instructions.
Will the output always be valid Lua syntax?
Yes. The web engine uses a tokenizer — it operates on individual tokens, never on raw text. Function keywords are preserved, string decrypt helpers are hoisted to the top, and XOR uses a Lua 5.1-compatible pure-Lua implementation.
What does "VM Runtime" toggle do?
When enabled, the output is wrapped in a VM runtime shell with an integrity checker and encrypted execution wrapper — matching the structure of the Python CLI's full bytecode output. Disable it for a lighter AST-only obfuscation.
Does it support Roblox Luau?
Yes. The XOR helper uses a pure-Lua fallback that works across Lua 5.1, LuaJIT, and Roblox Luau. The ~ operator is never used in output.
Can I obfuscate multiple files?
Each file is obfuscated independently. For full VM bytecode on multiple modules, run python cli.py on each file separately, then assemble in Roblox Studio as usual.
